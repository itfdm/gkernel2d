Введение
--------
Областью применения GKERNEL является окончательная проверка топологии на соответствие правилам топологического проектирования (Design RuleCheck, DRC) и проверка соответствия электрической схемы и топологии СнК (LayoutVersusSchematic, LVS).
Данное ПО позволяет вычислять характеристики интегральной схемы на этапе верификации конструктивно-технологических ограничений (КТО).

Назначение и условия применения программы
------------------------------------------

Структура ПО
~~~~~~~~~~~~
Основными подсистемами, обеспечивающими функционированиие ПО, являются:

1. `gkernel_structures <index.html#gkernel_structures>`__
2. `parser <index.html#parser>`__
3. `serializer <index.html#serializer>`__
4. `gkernel <index.html#gkernel>`__

Подсистема gkernel_structures
~~~~~~~~~~~~~~~~~~~~~~~~~~~

Подсистема "gkernel_structures" определяет ряд объектов, предназначенных для хранения входных данных, которые в дальнейшем подаются на вход ф-циям библиотеки `gkernel`,
а также для хранения выходных данных, которые генерируются в результате работы ф-ций библиотеки `gkernel`.

Подсистема parser
~~~~~~~~~~~~~~~~~

| Подсистема "parser" предназначена для разбора входных данных и перевод их во внутреннее представление системы, т.е. для создания и заполнения данными объектов подсистемы `gkernel_structures`.
| В качестве входных данных подсистема принимает путь до файла, который содержит описание контуров.

Подсистема serializer
~~~~~~~~~~~~~~~~~~~~~

| Подсистема "serializer" предназначена для перевода данных из внутреннего представления системы в текстовый формат.

Сообщения
---------

Подсистемы gkernel и gkernel_structures
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

* Подсистемы не генерирует никаких исключений не считая исключений, генерируемых стандартной библиотекой C++.
* Подсистемы не выводит никаких системных сообщений.

Подсистемы serializer и parser
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

* Подсистемы генерирует исключение `std::runtime_error` с сообщением: "Cannot |nbsp| open |nbsp| file |nbsp| *filename*" в случае невозможности открыть файл на чтение (или на запись).
